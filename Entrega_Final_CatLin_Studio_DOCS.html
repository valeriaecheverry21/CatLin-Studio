CatLin Studio

Entrega Final - Desarrollo de Web Estática

Proyecto: HTML5 + SASS + Bootstrap 5

Temática: Estudio de Belleza (Manicura, Cejas, Pestañas, Masajes)

Fecha: Mayo 2026

1. Introducción del Proyecto

Descripción General

Nombre del sitio web: CatLin Studio

Tipo de sitio: Institucional (Negocio ficticio de servicios de belleza)

Objetivo del sitio: Brindar una presencia web profesional para un estudio de belleza que ofrece servicios de manicura, diseño de cejas, extensiones de pestañas y masajes relajantes. El sitio permite a las clientas conocer los servicios, precios, equipo y contactar fácilmente.

Público objetivo: Mujeres de 18 a 55 años interesadas en cuidado personal, belleza y bienestar.

Descripción de la temática: CatLin Studio es un sitio web estático desarrollado con HTML5 semántico, SASS y Bootstrap 5. Cuenta con 5 secciones: Inicio, Servicios, Nosotros, Contacto y Galería. El diseño utiliza una paleta de colores suaves (rosas y tonos neutros) que transmiten elegancia y confianza. Se implementaron animaciones AOS para mejorar la experiencia de usuario.

Características HTML Implementadas

  • Etiquetas semánticas: <header>, <main>, <section>, <footer>, <nav>, <article>

Uso de Mixins (_mixins.scss)

Los mixins permiten reutilizar bloques de código con o sin parámetros.

@mixin respond-to($breakpoint) {
  @if $breakpoint == tablet {
    @media (max-width: $bp-tablet) { @content; }
  } @else if $breakpoint == mobile {
    @media (max-width: $bp-mobile) { @content; }
  }
}

@mixin card-hover {
  @include transition(transform, 0.3s, ease);
  &:hover {
    transform: translateY(-8px);
  }
}

@mixin button-style($bg-color: $color-primary) {
  background-color: $bg-color;
  color: $color-white;
  padding: $spacing-sm $spacing-lg;
  border-radius: $radius-md;
  
  &:hover {
    background-color: color.adjust($bg-color, $lightness: -10%);
    transform: translateY(-2px);
  }
}
📸 Captura: scss/abstracts/_mixins.scss

Uso de Nesting (Anidación)

SASS permite anidar selectores para mantener el código organizado.

.navbar-custom {
  background-color: $color-white;
  padding: $spacing-sm 0;
  box-shadow: $shadow-sm;
  
  .navbar-brand {
    font-family: $font-secondary;
    color: $color-primary-dark;
    
    span { color: $color-dark; }
  }
  
  .nav-link {
    color: $color-text;
    position: relative;
    
    &::after {
      content: '';
      position: absolute;
      bottom: 0;
      left: $spacing-sm;
      right: $spacing-sm;
      height: 2px;
      background-color: $color-primary;
      transform: scaleX(0);
      @include transition(transform, 0.3s, ease);
    }
    
    &:hover::after {
      transform: scaleX(1);
    }
  }
}
📸 Captura: scss/components/_nav.scss (ejemplo de nesting)

Uso de Extend/Placeholders

Los placeholders permiten compartir estilos entre selectores sin duplicar código.

// Placeholder
%section-padding {
  padding: $spacing-xxl 0;
  
  @include respond-to(tablet) {
    padding: $spacing-xl 0;
  }
  
  @include respond-to(mobile) {
    padding: $spacing-lg 0;
  }
}

// Uso en páginas
.hero { 
  @extend %section-padding; 
  @include gradient-bg;
}

.services-preview { 
  @extend %section-padding; 
  background-color: $color-light;
}

4. Responsive Design y Estilos Avanzados

Adaptación a Diferentes Dispositivos

El sitio utiliza un enfoque mobile-first, donde el diseño base es para móviles y luego se adapta a tablets y desktop usando media queries a través de mixins SASS.

// Ejemplo de responsive en SASS
.grid-4 {
  grid-template-columns: repeat(4, 1fr);
  
  @include respond-to(desktop) { 
    grid-template-columns: repeat(2, 1fr); 
  }
  
  @include respond-to(mobile) { 
    grid-template-columns: 1fr; 
  }
}

// Tipografía responsiva
html {
  font-size: 16px;
  
  @include respond-to(tablet) { 
    font-size: 15px; 
  }
  
  @include respond-to(mobile) { 
    font-size: 14px; 
  }
}

Animaciones y Transiciones

Se implementaron tres tipos de animaciones:

  1. AOS (Animate On Scroll): Animaciones al hacer scroll (fade-up, fade-left, zoom-in)
  2. Transiciones CSS: Efectos hover en cards, botones y enlaces
  3. Transformaciones: transform: translateY(-8px) en cards al hacer hover
// Animación de flotación en el hero
@keyframes float {
  0%, 100% { transform: translateY(0px); }
  50% { transform: translateY(-20px); }
}

.hero::before {
  content: '';
  position: absolute;
  top: -50%;
  right: -50%;
  width: 100%;
  height: 100%;
  background: rgba($color-white, 0.1);
  border-radius: 50%;
  animation: float 6s ease-in-out infinite;
}

// Transición en botones
.btn-custom {
  @include transition(all, 0.3s, ease);
  
  &:hover {
    transform: translateY(-2px);
    box-shadow: $shadow-md;
  }
}

7. Reflexión Final

El desarrollo de CatLin Studio permitió aplicar de manera integral los conocimientos de HTML5 semántico, preprocesadores SASS y frameworks CSS como Bootstrap. La principal dificultad encontrada fue la configuración correcta de los módulos SASS (uso de @use en lugar de @import) y la compilación correcta respetando la arquitectura 7-1.

Otro desafío fue personalizar Bootstrap manteniendo la identidad visual propia, logrando un sitio que no parece un template genérico. La implementación de AOS para animaciones mejoró significativamente la experiencia de usuario, haciendo que el sitio se sienta dinámico y moderno.

El resultado es un sitio web institucional profesional, totalmente responsive y con un código mantenible gracias a la estructura de SASS. Se cumplieron todos los requisitos de la evaluación final.
